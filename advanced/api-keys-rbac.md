# API Keys & RBAC

> Manage API keys with role-based access control for multi-user deployments.

🚧 **This page is under construction.** Content coming soon.

## Overview

GoClaw supports API key authentication with three RBAC roles: **admin** (full access), **operator** (read+write), and **viewer** (read-only). API keys can be created, listed, and revoked through the REST API or WebSocket RPC.
